# 🪐 Inner Orbit: Engineering the Mind

**"Refactor your life, Debug your emotions."**

**Gravity Assist Protocol: AI 기반 감정 로그 애플리케이션**

Inner Orbit은 개발자의 복잡한 내면(Inner World)을 시스템적으로 관리하고, 파편화된 삶의 경험을 구조화된 데이터로 변환하는 멘탈 케어 & 인사이트 로깅 플랫폼입니다. 우주 비행의 메타포를 활용하여 감정 일지 및 자기 성찰을 제공하며, 사용자의 감정 상태를 "궤도 안정성(Stability)"과 "중력(Gravity)" 지표로 시각화하고, AI를 통해 심리적 패턴과 인지 왜곡을 분석합니다.

우리는 복잡한 분산 시스템을 다루듯, 우리의 마음과 일상에도 정교한 아키텍처가 필요하다고 믿습니다.

## 🚀 Evolution: From "Logging" to "Engineering"

단순히 하루를 기록하는 일기장을 넘어, Inner Orbit은 개발자의 관점에서 삶을 **디버깅(Debugging)**하고 **리팩토링(Refactoring)**하는 도구로 진화했습니다.

우리의 뇌라는 CPU가 과도한 프로세스로 인해 부하가 걸릴 때, 혹은 일상의 '스파게티 코드' 같은 사건들이 반복될 때, Inner Orbit은 이를 구조화하고 최적화할 수 있는 세 가지 핵심 궤도(Orbit)를 제공합니다.

## 🏗️ Core Modules (The 3 Orbits)

Inner Orbit은 세 가지 핵심 궤도를 제공하여 다양한 멘탈 케어 요구사항을 충족합니다:

### 1. 📊 Daily Log: System Monitoring

**"일상의 흐름과 전반적인 상태(Mood)를 주기적으로 체크합니다."**

- **Intent**: 시스템 가동 시간(Uptime) 동안 발생한 일반적인 이벤트를 추적하여 멘탈 시스템의 기저 상태(Baseline)를 모니터링합니다.
- **Mechanism**:
  - **안정성(Stability) & 중력(Gravity)** 지표를 통한 감정 상태 기록
  - 자유로운 텍스트 입력으로 일상과 감정 기록
  - AI 기반 Navigation Query로 작성 가이드 제공
  - 일반적인 텍스트 기반 기록과 감정 지수 매핑
- **Effect**:
  - **AI 인지 왜곡 분석**: 일기 내용에서 인지 왜곡 패턴을 감지하고 CBT 기반 피드백 제공
  - 멘탈 시스템의 전반적인 상태 파악
- **Theme**: 녹색(green) 🟢

### 2. 🌿 Sensory Log (Deep Log): System Stabilization

**"When the system creates infinite loops of anxiety, focus on the raw input."**

- **Intent**: 번아웃이나 과도한 몰입(Over-engineering)으로 CPU 사용량이 100%에 달했을 때, 복잡한 연산(걱정, 후회)을 강제 종료하고 **Raw I/O(감각)**에 집중합니다.
- **Mechanism**:
  - '생각'을 배제하고 **위치, 시각(Visual), 청각(Auditory), 촉각(Tactile)** 데이터를 수집
  - **위치 정보**: 장소 또는 여행지 (최대 500자)
  - **3가지 감각 정보** 각 최대 10,000자:
    - 👁️ 시각: 풍경, 색상, 빛의 느낌
    - 👂 청각: 소리, 분위기
    - ✋ 촉각: 온도, 바람, 질감
- **Effect**:
  - 런타임 환경을 '현재(Present)'로 강제 고정(Grounding)
  - 멘탈 프로세스의 부하를 즉시 낮추고 시스템 안정성 회복
  - 여행 일지, 명상 기록, 자연 체험 등에 활용
- **Theme**: 라임 그린(lime/emerald) 🍃

### 3. 💡 Insight Log: Architecture of Life

**"Refactor your daily chaos into structured patterns."**

- **Intent**: 무질서한 사건들을 CS(Computer Science) 개념으로 추상화하여 삶의 지혜로 재사용(Reuse)합니다.
- **Mechanism - 3단계 구조**:
  1. **Trigger (Observation)**: 일상에서 관찰한 특정 현상 기록 (예: 데이터베이스 데드락 발생)
  2. **Abstraction (CS Concept)**: 현상을 기술적 개념으로 치환 (예: 자원 경합, 상호 배제, Race Condition, Load Balancer)
  3. **Application (Solution)**: 삶에 적용할 알고리즘 도출 (예: "내 일정도 우선순위 큐(Priority Queue)가 필요하다")
- **AI Features**:
  - **키워드 추천**: 관찰 내용 기반 CS 개념 자동 제안
  - **AI 피드백**: 3단계 내용을 종합하여 심화 학습 제안 및 피드백 제공
- **Effect**:
  - 경험이 **구조화된 데이터(Wisdom Library)**로 저장
  - 향후 유사한 문제 발생 시 최적의 솔루션을 제공하는 자산이 됨
- **Theme**: 청록색(teal/cyan) 🔷

## ✨ Key Features

### 🤖 AI Mental Analyst

Inner Orbit은 OpenAI GPT-4o-mini를 활용하여 작성된 로그를 분석하고, 심리적 통찰과 기술적 인사이트를 제공합니다.

#### 🧭 Navigation Query (항해 질문)
- **Static Mode**: 사전 정의된 6가지 질문 제공
- **AI Request Mode**: Gravity/Stability 상태 기반 동적 질문 생성
  - 현재 감정 상태에 맞춤화된 질문 자동 생성
  - 탭 클릭 시 자동 요청으로 UX 개선

#### 🧠 Cognitive Distortion Analysis (인지 왜곡 분석)
- Daily Mode 로그에 대한 AI 분석 제공
- 인지행동치료(CBT) 기반 왜곡 패턴 감지
- 재구성(Reframed) 및 대안적 관점(Alternative) 제시
- OpenAI GPT-4o-mini 모델 사용

#### 💬 Insight Feedback (통찰 피드백)
- Insight Mode 로그에 대한 AI 피드백
- 관찰-개념-적용의 연결성 평가
- CS 개념 심화 학습 방향 제안
- 실무 적용 가능성 검토

#### 🔑 CS Keyword Recommendation
- 관찰 내용 기반 관련 CS 개념 자동 제안
- 인지 왜곡(Cognitive Distortion) 감지

### 📊 Visualization & Analytics (시각화 & 분석)

- **🛸 Flight Trajectory**: 시간에 따른 감정 궤적 그래프 (Recharts)
  - 데이터가 쌓일수록 당신만의 '내면 우주(Inner Universe)'가 형성됩니다
- **📈 Analytics Dashboard**: 안정성, 중력 분포 및 트렌드 분석
- **🔍 Pattern Analysis**: 인지 왜곡 패턴 분석 및 통계
- **📅 ISO 8601**: 표준 날짜 형식으로 데이터 일관성 보장

### 💾 Data Management (데이터 관리)

- **📜 History**: 로그 히스토리 조회, 수정, 삭제
- **📥 Import/Export**: JSON 형식 데이터 내보내기/가져오기
- **🗄️ PostgreSQL**: 기반 영구 저장 및 JSONB 지원
- **♾️ Infinite Scroll**: 무한 스크롤 페이지네이션
- **🕐 Real-time Updates**: JPA flush()를 통한 즉각적인 updatedAt 반영
- **🌐 Multi-Mode System**: V5/V6 데이터 마이그레이션을 통해 DAILY, SENSORY, INSIGHT 모드를 완벽히 분리하여 목적에 맞는 데이터 스키마 제공

## 🚀 Getting Started

### 📋 Prerequisites (사전 요구사항)

- **Java 21** 이상 ☕
- **Node.js 18** 이상 📗
- **PostgreSQL 14** 이상 🐘
- **OpenAI API Key** 🔑

### ⚙️ Installation & Setup (설치 및 실행)

#### 🐳 Method 1: Docker Compose (권장)

가장 빠르고 간편한 방법입니다.

**1. 저장소 클론**

```bash
git clone <repository-url>
cd inner-orbit-system
```

**2. 환경 변수 설정**

프로젝트 루트에 `.env` 파일을 생성하고 다음 내용을 추가합니다:

```env
# Database Configuration
DB_NAME=innerorbit_db
DB_USER=postgres
DB_PASSWORD=your_secure_password

# OpenAI API Key (필수)
OPENAI_API_KEY=sk-proj-your-api-key-here
```

**3. Docker Compose 실행**

```bash
docker-compose up -d
```

이제 다음 주소에서 애플리케이션에 접근할 수 있습니다:
- 프론트엔드: `http://localhost:5173`
- 백엔드 API: `http://localhost:8080`
- PostgreSQL: `localhost:5432`

**4. 중지 및 재시작**

```bash
# 중지
docker-compose down

# 재시작
docker-compose up -d
```

---

#### 방법 2: 로컬 개발 환경

**1. 저장소 클론**

```bash
git clone <repository-url>
cd inner-orbit-system
```

**2. 데이터베이스 설정**

PostgreSQL에 데이터베이스를 생성합니다:

```sql
CREATE DATABASE innerorbit_db;
```

**3. 백엔드 설정**

환경 변수를 설정하거나 `backend/src/main/resources/application.yaml`을 수정합니다:

```yaml
spring:
  datasource:
    url: ${SPRING_DATASOURCE_URL:jdbc:postgresql://localhost:5432/innerorbit_db}
    username: ${SPRING_DATASOURCE_USERNAME:postgres}
    password: ${SPRING_DATASOURCE_PASSWORD:postgres}

  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true

  ai:
    openai:
      api-key: ${OPENAI_API_KEY:your_openai_api_key}
      chat:
        options:
          model: gpt-4o-mini
```

백엔드 서버 실행:

```bash
cd backend
./gradlew bootRun
```

**4. 프론트엔드 설정**

의존성 설치:

```bash
cd frontend
npm install
```

프론트엔드 개발 서버 실행:

```bash
npm run dev
```

애플리케이션이 `http://localhost:5173`에서 실행됩니다.

## 🔌 API Endpoints

### 📝 Log Entries

- `GET /api/logs` - 모든 로그 엔트리 조회 (최신순)
- `POST /api/logs` - 새 로그 엔트리 생성
- `GET /api/logs/{id}` - 특정 로그 조회
- `PUT /api/logs/{id}` - 로그 엔트리 수정
- `DELETE /api/logs/{id}` - 로그 엔트리 삭제
- `POST /api/logs/{id}/analyze` - AI 기반 인지 왜곡 분석 (Daily Mode)
- `POST /api/logs/{id}/insights/feedback` - AI 피드백 생성 (Insight Mode)
- `GET /api/logs/statistics` - 통계 조회

### 🤖 AI Services

- `GET /api/ai/prompt?gravity={0-100}&stability={0-100}` - 동적 Navigation Query 생성
  - **설명**: 사용자의 현재 Gravity/Stability 상태에 맞는 맞춤형 질문 생성
  - **파라미터**:
    - `gravity`: 외부 인력 (0-100, 기본값: 50)
    - `stability`: 코어 안정성 (0-100, 기본값: 50)
  - **응답**: `{"prompt": "생성된 한국어 질문"}`

- `POST /api/ai/insights/suggest-keywords` - Insight Mode AI 키워드 추천
  - **설명**: 관찰 내용을 기반으로 관련 CS 개념 자동 제안
  - **요청 본문**: `{"trigger": "관찰 내용"}`
  - **응답**: `{"keywords": ["RaceCondition", "ConcurrencyControl", ...]}`

## 🗄️ Database Schema

### 📊 LogEntry Table

**기본 필드**:
- `id` (BIGINT, Primary Key)
- `content` (TEXT) - 로그 내용
- `stability` (INTEGER, 0-100) - 안정성 지표
- `gravity` (INTEGER, 0-100) - 중력 지표
- `created_at` (TIMESTAMP) - ISO 8601 형식
- `updated_at` (TIMESTAMP) - ISO 8601 형식
- `user_id` (BIGINT)
- `analysis_result` (JSONB) - AI 분석 결과

**Sensory Mode 필드**:
- `location` (VARCHAR 500) - 여행지/장소
- `sensory_visual` (TEXT) - 시각 정보
- `sensory_auditory` (TEXT) - 청각 정보
- `sensory_tactile` (TEXT) - 촉각 정보

**Insight Mode 필드**:
- `insight_abstraction` (TEXT) - CS 개념 연결
- `insight_application` (TEXT) - 실무 적용
- `ai_feedback` (TEXT) - AI 피드백 결과

**Log Type**:
- `log_type` (VARCHAR 20) - DAILY, SENSORY, INSIGHT
- `is_deep_log` (BOOLEAN, deprecated) - 하위 호환성 유지용

모든 모드별 필드는 nullable로 설계되어 세 가지 모드 간의 호환성을 보장합니다.

## 🔒 Security & Architecture

### 🔑 OpenAI API Key Management

이 프로젝트는 **보안 강화**를 위해 OpenAI API 키를 백엔드에서 관리합니다:

- 🔐 API 키는 백엔드 환경 변수(`OPENAI_API_KEY`)로 관리
- 🚫 프론트엔드에서 API 키가 노출되지 않음
- 🔌 모든 AI 기능(분석, 프롬프트 생성, 키워드 추천, 피드백)은 백엔드 REST API를 통해 제공
- 💾 `localStorage`에 민감 정보 저장하지 않음

### 🏛️ Architecture Highlights

- **🏗️ Layered Structure**: Controller → Service → Repository → Entity
- **📦 DTO Pattern**: 요청/응답 데이터 분리로 유연성 향상
- **⚠️ Global Exception Handling**: 통일된 에러 응답 형식
- **🧪 Integration Testing**: H2 인메모리 DB로 데이터베이스 레벨 검증
- **🔄 Flyway Migration**: 데이터베이스 스키마 버전 관리 (데이터 아키텍처 진화)
- **📅 ISO 8601 Standard**: 국제 표준 날짜 형식으로 시간대 문제 해결
- **⚡ JPA Flush Optimization**: 즉각적인 엔티티 상태 반영
